## 🌟 Ways to Contribute

You can contribute in multiple ways:

- 🐛 Report bugs

- 📖 Improve documentation
- ⚡ Optimize workflow performance
- 🤖 Enhance AI email generation
- 🔌 Add new integrations (APIs, tools, services)

---

## ⚙️ Getting Started

### 1️⃣ Fork the Repository
Click the **Fork** button on the top-right corner of the repo page.

### 2️⃣ Clone Your Fork

```bash
git clone https://github.com/nikhil-ai-insights/<your-repo-name>.git
cd <your-repo-name>
```

### 3️⃣ Create a Branch

```bash
git checkout -b feature/your-feature-name
```

---

## 🛠️ Development Guidelines

To maintain quality and consistency:

- Keep workflows clean, modular, and readable
- Use clear node naming conventions in n8n
- Avoid breaking existing functionality
- Reuse nodes wherever possible
- Follow best practices for API usage

---

## 🤖 AI Contribution Guidelines

When working with AI features:

- Ensure prompts are clear and deterministic
- Avoid generating spammy or generic emails
- Focus on personalization and relevance
- Keep outputs professional and concise

---

## 🧪 Testing Checklist

Before submitting your contribution, make sure:

- ✅ Workflow executes without errors
- ✅ Google Sheets integration works correctly
- ✅ Emails are generated and sent properly
- ✅ No duplicate emails are triggered
- ✅ Rate limiting and delay logic works

---

## 📝 Commit Message Guidelines

Use clear and standardized commit messages:

```
feat: add AI-based subject line generation
fix: resolve duplicate email bug
docs: improve README clarity
refactor: optimize workflow structure
```

---

## 🚀 Submitting a Pull Request

1. Push your changes:

```bash
git push origin feature/your-feature-name
```

2. Open a **Pull Request** on GitHub.

3. In your PR, include:
   - What you changed
   - Why you changed it
   - Screenshots or workflow preview (if applicable)
