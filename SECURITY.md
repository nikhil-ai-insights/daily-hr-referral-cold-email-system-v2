## 🚨 Reporting a Vulnerability

If you discover a security vulnerability, please report it **responsibly and privately**.

### 📩 How to Report

- 📧 **Email:** *(add your email here)*
- 🐙 **GitHub:** [github.com/nikhil-ai-insights](https://github.com/nikhil-ai-insights)
- ⚠️ **Avoid** creating public issues for sensitive vulnerabilities

### 📝 What to Include in Your Report

Please provide as much detail as possible:

- Description of the vulnerability
- Steps to reproduce the issue
- Potential impact and severity
- Screenshots or proof-of-concept (if applicable)
- Suggested mitigation or fix *(optional)*

---

## ⏱️ Response & Resolution Timeline

We aim to:

- Acknowledge reports within **48–72 hours**
- Provide updates on progress
- Release a fix within **5–10 days** (based on severity)

## 🔒 Security Best Practices

This project involves automation, AI, and email integrations. Please follow these guidelines:

### 🔑 Credentials & Secrets

- Never commit API keys, tokens, or credentials
- Use environment variables or n8n credentials manager
- Rotate keys regularly

### 📧 Email Usage

- Do not use this system for spam or phishing
- Follow applicable regulations (GDPR, CAN-SPAM, etc.)
- Respect recipient consent and privacy

### 📊 Data Protection

- Store only necessary data
- Avoid storing sensitive personal information
- Restrict access to Google Sheets

---

## ⚠️ Known Risks

- Misuse of automation for spam outreach
- Exposure of Gmail or API credentials
- Improper rate limiting leading to account restrictions

---

## 🧰 Recommended Security Setup

We strongly recommend:

**🔐 Enable 2FA (Two-Factor Authentication) on:**
- GitHub
- Gmail
- Google Cloud

**🔒 Use:**
- OAuth2 authentication
- Secure API connections
- Access-controlled Google Sheets

## 🚫 Out of Scope

The following are **NOT** considered security vulnerabilities:

- Feature requests
- UI/UX improvements
- Minor bugs without security impact
- Documentation updates
